# Reflection

## What was the problem you were solving in the projects for this course?

The primary problem I was solving in this course was determining the most effective data structure to store, organize, and retrieve course information for ABC University's advising system. The advising software needed to load course data from a file, display courses in alphanumeric order, and quickly provide detailed information about individual courses and their prerequisites. The projects required evaluating multiple data structures and implementing a working solution that met both performance and functionality requirements.

## How did you approach the problem? Consider why data structures are important to understand.

I approached the problem by analyzing and comparing vectors, hash tables, and binary search trees. I evaluated the runtime and memory usage of each structure and considered how each would perform when loading course data, searching for courses, and printing sorted course lists. Understanding data structures is important because the choice of structure directly affects the efficiency, scalability, and maintainability of a program. By comparing different approaches, I was able to select a binary search tree because it provided a good balance between efficient searching and maintaining sorted data.

## How did you overcome any roadblocks you encountered while going through the activities or project?

One of the main challenges I encountered was loading and accessing the course data file correctly in Visual Studio. I also experienced issues with user input and file paths when testing the program. I overcame these challenges by troubleshooting the project setup, testing individual sections of code, and carefully reviewing program output to identify the source of errors. Breaking larger problems into smaller steps made it easier to isolate and resolve issues.

## How has your work on this project expanded your approach to designing software and developing programs?

This project helped me better understand the importance of planning before coding. Creating pseudocode, analyzing runtime complexity, and evaluating multiple data structures showed me how design decisions can significantly impact program performance. I learned that software development involves more than simply writing code; it also requires selecting appropriate algorithms and data structures to solve problems efficiently.

## How has your work on this project evolved the way you write programs that are maintainable, readable, and adaptable?

My work in this course reinforced the importance of writing organized and readable code. I used meaningful variable names, functions with specific responsibilities, and comments to improve clarity. Separating tasks such as loading data, searching, and printing information into individual functions made the program easier to understand and modify. These practices improve maintainability and adaptability because future changes can be made without rewriting large portions of the program.
